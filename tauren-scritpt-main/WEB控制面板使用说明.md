
# 今日头条自动化脚本 - Web 控制面板使用说明

## 功能概述

Web 控制面板提供了一个友好的图形界面，用于实时监控和控制今日头条自动化脚本的运行。无需使用命令行即可轻松配置和运行脚本。

## 功能特性

- **实时状态监控**：查看脚本运行状态、当前循环次数等
- **统计数据展示**：总处理链接数、成功点赞数、失败点赞数、成功率
- **一键控制**：启动/停止脚本
- **在线配置**：可视化编辑所有配置参数
- **账号管理**：添加、编辑、删除账号
- **实时日志**：查看脚本运行日志
- **美观界面**：现代化的设计风格

## 快速开始

### 1. 安装依赖

确保已安装所有依赖：

```bash
pip install -r requirements.txt
```

### 2. 启动 Web 服务器

在项目根目录下运行：

```bash
python web_server.py
```

### 3. 访问控制面板

打开浏览器，访问：

```
http://localhost:5000
```

## 界面说明

### 运行状态卡片

- **运行状态**：显示脚本当前是"运行中"还是"已停止"
- **启动时间**：显示脚本启动的时间
- **当前循环**：显示当前执行到第几次循环 / 总循环次数

### 控制按钮

- **▶ 启动脚本**：点击启动自动化脚本
- **⏹ 停止脚本**：点击停止正在运行的脚本

### 统计数据

- **总处理链接**：脚本运行以来处理的链接总数
- **成功点赞**：成功点赞的数量
- **失败点赞**：点赞失败的数量
- **成功率**：成功点赞占总点赞的百分比

### 系统配置

在系统配置区域可以编辑以下参数：

#### 基础配置

- **最小延迟(秒)**：操作之间的最小延迟时间
- **最大延迟(秒)**：操作之间的最大延迟时间
- **最大重试次数**：操作失败时的最大重试次数
- **每次处理推荐内容数**：每次循环处理的推荐内容数量

#### 浏览器配置

- **无头模式(后台运行)**：是否在后台运行浏览器（不显示界面）
- **最大化窗口**：是否最大化浏览器窗口

#### 循环配置

- **最大循环次数**：脚本执行的总循环次数
- **循环间隔(分钟)**：两次循环之间的等待时间
- **每次处理用户数**：每次循环处理的关注用户数量
- **每个用户处理视频数**：每个用户处理的视频数量

#### 日志配置

- **日志级别**：DEBUG / INFO / WARNING / ERROR
- **日志目录**：日志文件保存目录

### 账号管理

- **添加账号**：点击"➕ 添加账号"按钮添加新账号
- **编辑账号**：点击账号卡片上的"编辑"按钮
- **删除账号**：点击账号卡片上的"删除"按钮

账号信息包括：
- 账号名称
- Cookie 文件
- 账号状态
- 操作次数
- 最后使用时间

### 运行日志

实时显示脚本运行的日志信息，包括：
- 时间戳
- 日志级别（INFO/ERROR/WARNING）
- 日志消息

## API 接口

### GET /

访问控制面板主页

### GET /api/status

获取脚本当前状态

**响应示例：**
```json
{
  "running": false,
  "start_time": "2026-03-10 15:30:00",
  "current_cycle": 2,
  "total_cycles": 5,
  "stats": {
    "total_links": 20,
    "success_likes": 18,
    "failed_likes": 2,
    "failed_reasons": {}
  },
  "logs": [...]
}
```

### POST /api/start

启动脚本

**响应示例：**
```json
{
  "success": true,
  "message": "脚本已启动"
}
```

### POST /api/stop

停止脚本

**响应示例：**
```json
{
  "success": true,
  "message": "停止信号已发送"
}
```

### GET /api/config

获取配置信息

### POST /api/config

更新配置

**请求示例：**
```json
{
  "operation": {
    "min_delay": 2,
    "max_delay": 4,
    "max_retries": 2,
    "max_links_per_cycle": 10,
    "browser": {
      "headless": false,
      "maximize": true
    },
    "cycle": {
      "max_cycles": 5,
      "cycle_interval": 3600,
      "users_per_cycle": 10,
      "videos_per_user": 2
    }
  },
  "logging": {
    "level": "INFO",
    "log_dir": "logs"
  }
}
```

### GET /api/accounts

获取账号列表

### POST /api/accounts

添加账号

### PUT /api/accounts/&lt;account_id&gt;

更新账号

### DELETE /api/accounts/&lt;account_id&gt;

删除账号

## 注意事项

1. **端口占用**：默认使用 5000 端口，如需修改请编辑 `web/app.py` 或 `web_server.py`
2. **脚本运行**：Web 控制面板启动后，脚本会在后台线程中运行
3. **日志保留**：界面最多显示最近 100 条日志
4. **浏览器兼容性**：推荐使用 Chrome、Firefox、Edge 等现代浏览器
5. **配置生效**：修改配置后需要重启脚本才能生效

## 技术栈

- **后端**：Flask (Python Web 框架)
- **前端**：原生 HTML/CSS/JavaScript
- **样式**：渐变设计、响应式布局

## 常见问题

### Q: 如何修改默认端口？

A: 编辑 `web/app.py` 或 `web_server.py` 文件中的端口号。

### Q: 配置修改后多久生效？

A: 配置保存后立即生效，但正在运行的脚本需要重启才能使用新配置。

### Q: 日志文件保存在哪里？

A: 默认保存在 `logs/` 目录下，文件名格式为 `toutiao_YYYY-MM-DD.log`。

### Q: 可以同时运行多个脚本实例吗？

A: 不建议，Web 控制面板同一时间只能控制一个脚本实例。
