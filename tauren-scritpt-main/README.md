# TAUREN-SCRIPT

今日头条自动化脚本

## 项目简介

本项目是一个基于 Python 的今日头条自动化脚本，支持批量点赞、评论等自动化操作，提供功能完善的 Web 控制面板进行可视化管理，内置风控规避机制，操作安全稳定。

## ✨ 功能特性

### 核心功能
- **智能内容处理**：自动识别并处理推荐列表中的视频和文章内容
- **概率控制**：支持自定义点赞率、评论率，模拟真实用户行为
- **关注用户批量处理**：自动获取关注列表，批量处理关注用户的内容
- **多轮循环执行**：支持配置循环次数和间隔时间，全自动运行
- **多账号管理**：支持多账号切换，账号状态自动管理

### Web 控制面板
- **可视化监控**：实时查看运行状态、操作统计、实时日志
- **在线配置**：所有参数支持网页端配置，实时生效无需重启
- **日志系统**：支持日志搜索、级别过滤、一键清空
- **暗黑模式**：支持亮色/暗色主题切换，护眼友好
- **响应式设计**：完美适配桌面端和移动端

### 安全特性
- **随机延迟**：所有操作之间随机延迟，模拟人类操作
- **代理池支持**：集成代理池功能，自动轮换IP降低风控风险
- **行为模拟**：随机滚动、随机停留、真实操作轨迹
- **异常处理**：完善的错误捕获和重试机制，脚本稳定运行不崩溃

### 其他特性
- **Cookie自动保存**：登录一次后自动保存，后续无需重复扫码
- **数据统计**：实时显示点赞成功率、操作数量等统计信息
- **Toast提示**：友好的操作反馈，替代原始alert弹窗
- **账号风控检测**：自动检测账号状态，异常时及时提醒

## 🛠️ 技术栈

- **Python** 3.8+
- **Helium** - 简化网页自动化操作
- **Selenium** - 浏览器自动化驱动
- **Flask** - 轻量级Web框架
- **Edge浏览器** - 官方推荐浏览器（兼容性最好）
- **HTML/CSS/JavaScript** - 前端控制面板

## 🚀 安装说明

### 1. 环境要求
- Python 3.8 或更高版本
- Microsoft Edge 浏览器（系统自带即可，无需额外配置）
- Windows 操作系统

### 2. 安装依赖
```bash
pip install -r requirements.txt
```

### 3. 浏览器驱动
脚本会自动下载匹配的 EdgeDriver，无需手动配置。

## 📖 使用说明

### 方式一：Web 控制面板（强烈推荐）
1. 启动 Web 服务器：
```bash
python web/app.py
```

2. 打开浏览器访问：http://localhost:5000

3. 首次使用：
   - 在账号管理页面添加账号
   - 配置系统参数（操作概率、延迟、循环设置等）
   - 点击"启动脚本"按钮，首次运行会弹出浏览器，扫码登录今日头条即可
   - 后续运行会自动使用保存的Cookie登录

### 方式二：命令行运行
```bash
python main.py
```
脚本会使用 `config/config.yaml` 中的配置自动运行。

## ⚙️ 配置说明

所有配置项均可在 Web 控制面板中修改，实时生效。配置文件位于 `config/config.yaml`：

### 操作配置
| 配置项 | 说明 | 默认值
---------|------|-------
operation.min_delay | 操作最小延迟(秒) | 2
operation.max_delay | 操作最大延迟(秒) | 4
operation.max_retries | 操作失败最大重试次数 | 2
operation.max_links_per_cycle | 每次循环处理推荐内容数量 | 10
operation.like_rate | 点赞概率(0-100)% | 100
operation.comment_rate | 评论概率(0-100)% | 100

### 循环配置
| 配置项 | 说明 | 默认值
---------|------|-------
operation.cycle.max_cycles | 最大循环次数 | 5
operation.cycle.cycle_interval | 循环间隔(秒) | 3600
operation.cycle.users_per_cycle | 每次循环处理关注用户数量 | 10
operation.cycle.videos_per_user | 每个用户处理视频数量 | 5

### 浏览器配置
| 配置项 | 说明 | 默认值
---------|------|-------
operation.browser.headless | 无头模式（后台运行，不显示浏览器窗口） | false
operation.browser.maximize | 启动时最大化浏览器窗口 | true
operation.browser.proxy_enabled | 启用代理池 | false
operation.browser.proxy_pool | 代理池列表（每行一个代理） | []

### 日志配置
| 配置项 | 说明 | 默认值
---------|------|-------
logging.level | 日志级别（DEBUG/INFO/WARNING/ERROR） | INFO
logging.log_dir | 日志文件保存目录 | logs

## 💬 评论配置

`match_comment.text` 文件用于配置关键词和对应的评论内容，格式：
```
关键词>评论内容
```

示例：
```
精彩>说得太好了，非常认同你的观点！
学习>学习了，感谢分享这么优质的内容！
支持>支持一下，内容很棒！
加油>加油，期待你更多的作品！
```

脚本会根据页面标题的关键词匹配对应的评论内容，自动生成合适的评论。

## 🎯 代理池使用说明

### 1. 开启代理功能
在Web控制面板→系统配置→浏览器配置中，打开"启用代理池"开关。

### 2. 配置代理
在"代理池"输入框中每行填写一个代理地址，支持以下格式：
```
http://192.168.1.100:8080
socks5://127.0.0.1:1080
https://username:password@proxy-address:port
```

### 3. 代理特性
- 每次启动浏览器自动随机选择一个代理
- 支持HTTP/HTTPS/SOCKS5协议
- 可以有效降低同一IP操作频繁导致的风控风险

## ⚠️ 注意事项

1. **首次登录**：首次运行需要扫码登录今日头条，Cookie会自动保存到 `toutiao_cookies.pkl` 文件，后续无需重复登录
2. **合理设置参数**：建议延迟设置不小于2秒，点赞率和评论率不要设置为100%，避免被检测为机器人
3. **网络环境**：运行时确保网络稳定，避免频繁断连导致操作失败
4. **风险提示**：本工具仅供学习交流使用，使用时请遵守平台规则，请勿用于非法用途
5. **数据安全**：请妥善保管Cookie文件和账号信息，不要泄露给他人

## 🔧 常见问题

### Q: 运行时提示浏览器启动失败？
A: 确保已安装最新版Edge浏览器，或者检查网络是否正常。

### Q: 无法获取推荐内容？
A: 检查是否已成功登录，Cookie是否有效，尝试删除 `toutiao_cookies.pkl` 文件重新登录。

### Q: 点赞/评论失败？
A: 可能是页面结构更新导致定位器失效，或者操作过于频繁被限制，适当调大延迟参数。

### Q: Web控制面板无法访问？
A: 检查5000端口是否被占用，或者尝试修改 `web/app.py` 中的端口号。
